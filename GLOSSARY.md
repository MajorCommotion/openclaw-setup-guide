## DO NOT TRANSLATE

**Keep these exactly as-is in English:**

- Command names: `docker`, `git`, `ssh`, `curl`, `wget`, etc.
- Code blocks: All code stays in English
- URLs: Keep English URLs
- File paths: Keep English file paths (unless Korean localized)
- Environment variables: `VARIABLE_NAME`
- Configuration keys: `config.yaml` field names
- Error messages: Keep English error messages (add Korean explanation after if needed)

## Korean Typography Rules / 한국어 타이포그래피 규칙

**Spacing:**
- Put spaces around English words embedded in Korean text
- ✅ Correct: `OpenClaw는 멀티 에이전트 플랫폼입니다.`
- ❌ Wrong: `OpenClaw는멀티에이전트플랫폼입니다.`

**Punctuation:**
- Use Korean punctuation: `,` `.` `!` `?` (same as English)
- Use Korean quotes: `「」` or `""`
- Use Korean parentheses: `()` or `（）`

**Numbers:**
- Use Arabic numerals: `1, 2, 3`
- Large numbers: `1,000` or `1000` (both acceptable)
